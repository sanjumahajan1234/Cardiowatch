Privacy & Data Management

HIPAA-aligned data practices for CardioWatch — synthetic demo environment

FHIR Resources Accessed

CardioWatch reads the following FHIR R4 resource types from the HAPI public demo server. No write operations are performed against the server.

Resource Fields Used Purpose
Patient name, birthDate, gender, identifier Display demographics on dashboard and detail pages
Observation LOINC 85354-9, components 8480-6 / 8462-4 (BP), 8867-4 (HR) AHA/ACC classification, trend analysis, time-series chart
Condition code.text, clinicalStatus Display active diagnoses on patient detail page
MedicationRequest medicationCodeableConcept.text, status Display current medications on patient detail page
Security Practices
  • No real PHI stored or transmitted. All patient data is Synthea-generated synthetic data. No real patient identifiers are used at any stage.
  • FHIR server access over HTTPS. All queries to hapi.fhir.org are made via HTTPS. The application performs read-only GET requests only.
  • Email alerts sent over TLS (SMTP). Alert emails are delivered via Gmail SMTP with TLS encryption. Alerts contain only synthetic patient identifiers.
  • In-memory only — no persistent database in demo mode. Patient data is held in memory for the duration of the session and is not written to disk or any external store.
  • Password-gated clinical access. All patient-facing routes require session authentication. Sessions are cleared on logout and do not persist across server restarts.
Data Export & Deletion

Because CardioWatch operates exclusively on synthetic data in demo mode, there is no real patient data to delete. The export endpoint below provides a full JSON snapshot of the current in-memory dataset — useful for audit, grading, or offline inspection. Restarting the server clears all in-memory state.

Export Dataset (JSON) Returns all patient records, observations, conditions, and medications as a structured JSON file.
Synthetic Data Notice

All data displayed in this application was generated using Synthea with the hypertension and cardiovascular disease modules enabled. No real patient records were used at any stage of development or testing. This application is a course project prototype and is not intended for clinical deployment.
